JavaScript中async与defer属性的区别

Posted on 2016-10-09   |   In JavaScript

async与defer的区别
从上面这张图可以发现：

  • <script src="script.js"></script>
    浏览器在读到 script标签后，就开始加载、执行script标签中的代码，此时html解析暂停。
  • <script defer src="script.js"></script>
    脚本加载和html文档解析是并行进行的，但是script脚本的执行是在html解析完成后，DOMContentLoaded事件触发之前开始的。
    在浏览器加载并解析完html文档（样式表、图片、子窗口可能还没有加载完成），会触发DOMContentLoaded事件。
  • <script async src="myscript.js"></script>
    script脚本和html解析同步执行，并且script脚本在加载完成后立即执行，在脚本执行的这段时间内浏览器不会解析html文档。

jQuery中 bind delegate click on都是啥？

.bind( eventType [, eventData ], handler )

  • 给匹配到的元素绑定事件
  • 对于动态添加的元素，不会绑定事件。

.delegate( selector, eventType, eventData, handler )

  • 对于动态添加的元素，会绑定事件。

.click( handler )

  • 给元素绑定单击事件
  • 对于动态添加的元素，不会绑定事件。

.on( events [, selector ] [, data ], handler )

  • 对于动态添加的元素，会绑定事件
  • jQuery源码中.bind()、.delegate()方法都是通过.on()方法实现的。
如果用的jQuery版本是1.7+的话，建议使用.on()方法绑定事件。

jQuery中mouseover mouseout mouseenter mouseleave 的差别

mouseenter mouseleave

mouseenter、mouseleave是一对，是不冒泡事件，给子元素绑定mouseenter事件，当鼠标从外部移入子元素上的时候不会触发祖先元素的mouseenter事件。mouseleave是鼠标移出事件，也是不冒泡的。

mouseover mouseout

mouseover、mouseout是一对，是冒泡事件，给子元素绑定事件，当鼠标从外部移入子元素s上的时候，不仅会触发元素本身的mouseover事件，也会触发祖先元素的mouseover事件。mouseleave是鼠标移出事件，也是冒泡的。
